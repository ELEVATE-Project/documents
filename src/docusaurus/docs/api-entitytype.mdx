# Create Entity Types

In <PartialExample mentored /> service, entity types refer to the properties of sessions or mentors. The **label** of the entity types are displayed as fields on the profile creation page and session creation page.

For example:

  * Session properties include category and medium.

  * Mentor properties include role and expertise.

System administrators can create new entity types to achieve the following:

  * Define custom properties for sessions or mentors.

  * Add new text fields and select fields (such as single-select or multi-select fields) on the profile creation page or session details page.

  * Define new default rules. See <a href="api-defaultrules">Set Default Rules for an Organization</a> to learn more.

    <Admonition type="note">
    You can set default rules only for multi-select fields with a fixed range.
    </Admonition>

## Get Entity Types

To update or delete entity types, system administrators can obtain the details of entity types such as the ID and value.

Before setting default rules, organization administrators can obtain the **value** of the entity types.

:::note Prerequisites

Organization administrators must do the following actions:

  1. After receiving the application's link through email, sign up on the application. See the <a href="aboutorganizationadministrator">Organization Administrator</a> section to learn more.
  2. Log in using `{{url}}/user/v1/account/login` to generate an access_token.

:::
