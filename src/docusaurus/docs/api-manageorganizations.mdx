# Updating Organizations' Details

Organization administrators can update their organizations details such as name and description. Whereas, system administrators can configure the associated organizations for organizations with a **Associated** policy in addition to updating organizations' details. 

To learn more about the types of policies that can be configured, see [Policy Architecture](policyarchitecture#policy-architecture).

**HTTP Request**

Method: PATCH

```
{{url}}/user/v1/organization/update/{organization_id}

```

|Path Parameter|Data Type|Description|
|---|---|---|
|organization_id|Integer|Only system administrators should specify the ID of the organization for which they want to update the details. Optional.|

**Request Body**

<table>
<tr>
    <th>Payload Parameter</th>
    <th>Data Type</th>
    <th>Description</th>
</tr>
<tr>
    <td>name</td>
    <td><b>String</b></td>
    <td><p>Updated organization's name. Required.</p><p>Example: <i>ShikshaLokam</i></p></td>
</tr>
<tr>
    <td>description</td>
    <td><b>String</b></td>
    <td><p>Updated description of the organization. Required.</p><p>Example: <i>Empowering educational leadership.</i></p></td>
</tr>
<tr>
    <td>related_orgs</td>
    <td><b>Array of integers</b></td>
    <td><p>For organizations with an <b>Associated</b> policy, only system administrators can configure the associated organizations. Optional</p><p>Example: <i>"related_orgs": [3,4]</i></p>
    </td>
</tr>
</table>

**Request Header**

|Key|Value|Description|
|---|---|---|
|X-auth-token|bearer {{access_token}}|access_token received from the login response. Required.|

**Response**

If successful, this method returns a `200 OK` response code. If the organization does not exist, this method returns a `400 Bad Request` response.

## Deactivating Organizations

**HTTP Request**

Method: POST

  ```
  {{url}}/user/v1/admin/deactivateOrg/{organization_id}

  ```

|Path Parameter|Data Type|Description|
|---|---|---|
|organization_id|Integer|<p>ID of the organization that you want to deactivate. Required.</p><p>Example: <i>[12]</i></p>|

**Request Body**

This method does not require a request body.

**Request Header**

|Key|Value|Description|
|---|---|---|
|X-auth-token|bearer {{access_token}}|access_token received from the login response. Required.|

**Response**

If successful, this method returns a `202 Accepted` response code. If the organization is already deleted, this method returns a `400 Bad Request` response.

<table>
  <tr>
    <th>Response Parameter</th>
    <th>Data Type</th>
    <th>Description</th>
  </tr>
  <tr>
    <td>responseCode</td>
    <td><b>String</b></td>
    <td>A success response.</td>
  </tr>
  <tr>
    <td>message</td>
    <td><b>String</b></td>
    <td>A success message.</td>
  </tr>
  <tr>
    <td>result</td>
    <td><b>Array of objects</b></td>
    <td><b>deactivated_users</b> key specifies the number of users at the organization. The user accounts are also deactivated.</td>
  </tr>
</table>

## Deactivating Users

If a user leaves the organization, the organization administrator can deactivate the user's account.

**HTTP Request**

Method: POST

```
{{url}}/user/v1/org-admin/deactivateUser

```
**Request Body**

|Body Parameter|Data Type|Description|
|---|---|---|
|id|Array of integers|<p>ID of users whose accounts you want to deactivate. Required.</p><p>Example: <i>[1,2]</i></p>|

**Request Header**

|Key|Value|Description|
|---|---|---|
|X-auth-token|bearer {{access_token}}|access_token received from the login response. Required.|

**Response**

If successful, this method returns a `202 Accepted` response code. If the user is already deleted, this method returns a `400 Bad Request` response code.
