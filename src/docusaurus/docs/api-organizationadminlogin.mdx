# Logging in to the Organization Administrator Account

After receiving the application's link through email, you must sign up on the application. See the [Signing up](aboutorganizationadministrator#signing-up) section to learn more.

For actions such as setting policies for an organization or deactivating users, organization administrators must log in to their account.

**HTTP Request**

Method: POST

  ```
  {{url}}/user/v1/account/login

  ```

**Request Body**

<table>
  <tr>
    <th>Payload Parameter</th>
    <th>Data Type</th>
    <th>Description</th>
    <th>Required</th>
  </tr>
  <tr>
    <td>email</td>
    <td><b>String</b></td>
    <td><p>The login email ID.</p><p>Example: "<i>peter@shikshalokam.com</i>"</p></td>
    <td>True</td>
  </tr>
  <tr>
    <td>password</td>
    <td><b>String</b></td>
    <td><p>The login password.</p><p>Example: "<i>Password@123</i>"</p></td>
    <td>True</td>
  </tr>
</table>

**Response**

If successful, this method returns a `200 OK` response code. If the email or password is invalid, this method returns a `400 Bad Request` response code.
